# Predicting-Covid-19-using-Machine-Learning-Models
Project Overview This project aims to develop and evaluate various machine learning models for the prediction of SARS-CoV-2 (COVID-19) exam results. Utilizing a comprehensive patient dataset, the goal is to identify effective classification algorithms capable of accurately determining a patient's COVID-19 status, which can assist in early diagnosis and resource allocation. Dataset The dataset used in this project contains patient information, including demographic details (e.g., age quantile) and a wide range of laboratory test results (e.g., haematocrit, haemoglobin, platelets, leukocytes, and lymphocytes). The target variable is the SARS-CoV-2 exam result, which indicates either 'positive' or 'negative'. 
Key Data Characteristics: Original Size: 5644 entries, 111 columns. Class Imbalance: A significant challenge identified was the severe class imbalance, with approximately 9.89% positive cases and 90.11% negative cases. Missing Values: Many laboratory test result columns contained a high percentage of missing values. Methodology The project followed a structured machine learning pipeline: 
1. Data Loading and Initial Exploration Loaded data from dataset 2.xlsx into a Pandas DataFrame. Performed initial checks (df.head(), df.info(), df.describe()) to understand data structure, types, and summary statistics.
2. Data Preprocessing Column Cleaning: standardised column names (removed spaces, converted to lowercase). Missing Value Imputation: For selected features (e.g., haematocrit, haemoglobin, platelets), missing values were imputed using the mean strategy. Outlier Handling: Outliers were identified and removed using the Interquartile Range (IQR) method for key numerical features to assess their impact on model performance. Target Encoding: The SARS-Cov-2 exam result was mapped to a numerical target variable (covid_positive: 0 for negative, 1 for positive).
3. Feature Engineering Lymphocyte Ratio: Created lymphocyte_ratio by dividing lymphocytes by leukocytes. Age Group Categorisation: patient age quantile was binned into age_group categories (Young, Adult, Senior, Elderly). Platelet Transformation: Applied a log transformation to platelets (log_platelets) after handling zero values. One-Hot Encoding: The age_group categorical feature was One-Hot Encoded. Polynomial Features: Polynomial features (degree 2) were generated from the numerical features to capture non-linear relationships and interactions.
4. Handling Class Imbalance (SMOTE) Given the significant class imbalance, the Synthetic Minority Oversampling Technique (SMOTE) was applied to the training data to balance the classes and prevent models from being biased toward the majority class.
5. Model Selection and Training A diverse set of classification models were implemented and evaluated: Logistic Regression Decision Tree Classifier Random Forest Classifier Support Vector Machine (SVM) K-Nearest Neighbors (KNN) XGBoost Classifier LightGBM Classifier Multi-layer Perceptron (MLPClassifier) Ensemble Models: Voting Classifier: Combined predictions from multiple individual models. Stacking Classifier: Trained a meta-learner on the predictions of several base learners.
6. Model Evaluation Models were evaluated using a combination of metrics and techniques: Cross-Validation: StratifiedKFold (5-fold) was used to ensure robust evaluation and account for class imbalance during splitting. Performance Metrics: Accuracy, Precision, Recall, and F1-score were calculated. ROC AUC Score: The Receiver Operating Characteristic Area Under the Curve was used to assess the models' ability to distinguish between classes. Confusion Matrices: Visualise true positives, true negatives, false positives, and false negatives for each model. Hyperparameter Tuning: GridSearchCV was employed for some models to determine optimal hyperparameters.
Key Findings:
Effectiveness of SMOTE: SMOTE significantly improved the recall for the minority (positive) class across most models, demonstrating its importance in imbalanced medical datasets.
Impact of Outlier Handling: While outlier removal sometimes boosted overall accuracy, it occasionally led to a decrease in recall for the positive class, suggesting a need for careful consideration.
Top-Performing Models:
The Ensemble Stacking Classifier emerged as the strongest performer, achieving an accuracy of 97.4%, a precision of 97% for the positive class, and a recall of 99% for the positive class, with an F1-score of 98% and an ROC AUC of ~0.97 (approximated from similar ensemble performance).
Logistic Regression also showed strong performance with an ROC AUC of ~0.83 and balanced precision/recall.
Feature Importance: Features like patient age quantile, lymphocyte_ratio, and hematocrit consistently ranked high in importance across tree-based models, indicating their strong correlation with COVID-19 diagnosis.
